Objectif

Apprendre à manipuler des données Lidar, en extraire des informations d’intérêt pour modéliser des attributs forestiers liés à la structure des forêts, comme la surface terrière ou le volume.

La première partie du TD sera consacrée à la manipulation de nuages de points lidar. Nous exploiterons pour cela une tuile du programme lidar HD.

Dans un second temps, un travail sur la création de modèles numérique et la normalisation des nuages de point du sursol sera effectué.

La troisième partie portera sur l’extaction de descripteurs de la canopée, avec des approaches surfaciques et objet.

Finalement, la dernière partie du TD visera à construire un modèle prédictif d’attributs forestiers. Compte tenu de l’emprise spatiale des données, elle exploitera un fichier de données généré au préalable à partir de données IFN et d’un vol Lidar sur les Vosges.

  1. descripteurs objet

La fonction “segment_trees” de lidR permet de segmenter le nuage en couronnes d’arbre individuelles.

Elle inclut plusieurs algorithmes.

Dans l’exemple suivant, on utlise l’algorithme de Dalponte (2016). Les paramètres de l’algorithme sont - un CHM - un jeu de position d’arbre - plusieurs seuils qui contraigent la croissance de la couronnes)

Pour générer le jei de position d’arbre, on utilise un filtre de maxima locaux avec taille de fenêtre variable La taille de la fenêtre augmente avec la hauteur:

f <- function(x) {x * 0.1 + 3}
heights <- seq(0,30,5)
ws <- f(heights)

Ensuite, on localise les apex d’arbres. La fonction prend soit le nuage de point soit le CHM

apex <- locate_trees(clip_las_h, lmf(f))

On visualise le résultat

plot(chm, col = height.colors(50))
plot(sf::st_geometry(apex), add = TRUE, pch = 3)

On segmente ensuite le nuage de point par croissance de region depuis les maxima locaux

algo <- dalponte2016(chm, apex)
trees <- segment_trees(clip_las_h, algo)
plot(trees, bg = "white", size = 4, color = "treeID")

On peut raffiner les résultats en jouant sur la détection des maxima locaux ainsi que sur les paramètres de l’algorithme. Dans l’exemple suivant, on lisse le CHM avec un filtre median :

kernel <- matrix(1,3,3)
chm_smooth <- terra::focal(chm, w = kernel, fun = median, na.rm = TRUE)
apex <- locate_trees(chm_smooth, lmf(f))

plot(chm_smooth, col = height.colors(50))
plot(sf::st_geometry(apex), add = TRUE, pch = 3)

algo <- dalponte2016(chm_smooth, apex)
trees <- segment_trees(clip_las_h, algo)
plot(trees, bg = "white", size = 4, color = "treeID")

On compare les résultats avec un autre algorithme: li 2012 Cet algorithme traite directement le nuage de point et ne nécessite pas d’identification préalable de maxima locaux

algo <- li2012()
trees <- segment_trees(clip_las_h, algo)
plot(trees, bg = "white", size = 4, color = "treeID")

Partie 4 : modélisation d’attributs forestiers

Le jeux de données contient des données de volume et de surface terrière pour un ensemble de placettes IFN et des métriques lidar extraites pour chaque placette, sur une surface circulaire de 15 m de rayon. Les métriques lidar sont les suivantes: Perc10 à Perc95: des percentiles de hauteur, pour tous les points au dessus de 2 m Cum1 à Cum9 : les pourcentages cummulés de retours au dans la ieme couche (d1, d2 dans https://doi.org/10.5558/tfc84827-6)

Objectif : développer un modèle de prédiction du volume et/ou de la surface terrière par un modèle linéaire simple. Etapes : selectionner le meilleur modèle à partir de l’ensemble des modèles, appliquer une approche de validation croisé et de test.

  1. Lecture du fichier
plots <- read.csv("plots.csv")
head(plots)

Calcul des modèles pour le volume. On commence par supprimer la surface terrière du tableau

volume <- plots[,-c(3)]
models <- regsubsets(volume~., data = volume, nvmax = 5)
summary(models)

On considère ici le meilleur modèle à 3 variables : Cum1, Cum9, Hentropy

Que représentent ces variables selon vous?

On veut tester le modèle en validation croisée: k-fold

On commence par générer les échantillon et le vecteur pour stocker les erreurs

K=10 
folds = sample(1:K, nrow(volume), replace=TRUE)
err = rep(NA,K)

Ensuite, on itère pour chaque échantillon la construction du modèle, son évaluation sur les données non utilisées Et on calcule l’erreur quadratique moyenne

for (i in 1:K){
  # on selectionne l'échantillon k
  fold_i = which(folds==i)
  
  # on génère un jeu d'entrainement et de test
  volumeApp = volume[-fold_i,]
  volumeTest = volume[fold_i,]
  
  # on entraine le modèle linéaire
  App = lm(volume~., data=volumeApp)
  
  # on le teste sur le jeu d'évaluation
  Pred = predict(App, volumeTest)
  
  # on calcule l'erreur de prédiction et on l'assigne au vecteur de sortie
  err[i] = sqrt(mean((Pred - volumeTest$volume)^2))
}

# on calcule l'erreur moyenne et l'écart type de l'erreur sur les k folds
mean(err)
sd(err)

# on passe en pourcentage
mean(err)/mean(volume$volume)*100

Répéter l’opération avec différents modèles et trouver celui qui minimise l’erreur.
